## Installation

Faiss is distributed as precompiled conda packages. Choose the variant that matches your hardware.

<CodeGroup>

```bash CPU only
conda install -c pytorch -c conda-forge faiss-cpu=1.14.1
```

```bash GPU (CUDA)
conda install -c pytorch -c nvidia -c conda-forge faiss-gpu=1.14.1
```

```bash GPU with NVIDIA cuVS
conda install -c pytorch -c nvidia -c rapidsai -c conda-forge libnvjitlink faiss-gpu-cuvs=1.14.1
```

</CodeGroup>

<Note>
  The `conda-forge` channel is required for up-to-date BLAS dependencies (Intel MKL on x86-64, OpenBLAS on ARM). For `faiss-gpu`, the `nvidia` channel is additionally required for CUDA. See [INSTALL.md](https://github.com/facebookresearch/faiss/blob/main/INSTALL.md) for building from source with CMake.
</Note>

<Tip>
  `faiss-cpu` is available on Linux (x86-64 and aarch64), macOS (arm64), and Windows (x86-64). `faiss-gpu` requires Linux x86-64 with CUDA 11.4 or 12.1.
</Tip>

## Your first search

The following example builds an exact flat index (`IndexFlatL2`) over 100,000 random 64-dimensional vectors and searches for the 4 nearest neighbors of 10,000 query vectors.

<Steps>

<Step title="Generate some vectors">

Faiss works with 32-bit float arrays. Here we generate a database `xb` and a query set `xq`, both of dimension `d = 64`.

<Tabs>

<Tab title="Python">

```python
import numpy as np

d = 64                           # dimension
nb = 100000                      # database size
nq = 10000                       # nb of queries
np.random.seed(1234)             # make reproducible
xb = np.random.random((nb, d)).astype('float32')
xb[:, 0] += np.arange(nb) / 1000.
xq = np.random.random((nq, d)).astype('float32')
xq[:, 0] += np.arange(nq) / 1000.
```

</Tab>

<Tab title="C++">

```cpp
#include <cstdio>
#include <cstdlib>
#include <random>

#include <faiss/IndexFlat.h>

using idx_t = faiss::idx_t;

int main() {
    int d = 64;      // dimension
    int nb = 100000; // database size
    int nq = 10000;  // nb of queries

    std::mt19937 rng;
    std::uniform_real_distribution<> distrib;

    float* xb = new float[d * nb];
    float* xq = new float[d * nq];

    for (int i = 0; i < nb; i++) {
        for (int j = 0; j < d; j++)
            xb[d * i + j] = distrib(rng);
        xb[d * i] += i / 1000.;
    }

    for (int i = 0; i < nq; i++) {
        for (int j = 0; j < d; j++)
            xq[d * i + j] = distrib(rng);
        xq[d * i] += i / 1000.;
    }
```

</Tab>

</Tabs>

</Step>

<Step title="Build the index and add vectors">

`IndexFlatL2` performs exact L2 nearest-neighbor search. It requires no training — vectors can be added immediately.

<Tabs>

<Tab title="Python">

```python
import faiss                   # make faiss available
index = faiss.IndexFlatL2(d)   # build the index
print(index.is_trained)        # True — flat indexes need no training
index.add(xb)                  # add vectors to the index
print(index.ntotal)            # 100000
```

</Tab>

<Tab title="C++">

```cpp
    faiss::IndexFlatL2 index(d); // call constructor
    printf("is_trained = %s\n", index.is_trained ? "true" : "false");
    index.add(nb, xb);           // add vectors to the index
    printf("ntotal = %zd\n", index.ntotal);
```

</Tab>

</Tabs>

</Step>

<Step title="Search">

`index.search(queries, k)` returns two arrays: `D` (distances) and `I` (integer indices of nearest neighbors), both of shape `(nq, k)`.

<Tabs>

<Tab title="Python">

```python
k = 4                          # we want to see 4 nearest neighbors
D, I = index.search(xb[:5], k) # sanity check
print(I)
print(D)
D, I = index.search(xq, k)     # actual search
print(I[:5])                   # neighbors of the 5 first queries
print(I[-5:])                  # neighbors of the 5 last queries
```

</Tab>

<Tab title="C++">

```cpp
    int k = 4;

    { // sanity check: search 5 first vectors of xb
        idx_t* I = new idx_t[k * 5];
        float* D = new float[k * 5];

        index.search(5, xb, k, D, I);

        printf("I=\n");
        for (int i = 0; i < 5; i++) {
            for (int j = 0; j < k; j++)
                printf("%5zd ", I[i * k + j]);
            printf("\n");
        }

        printf("D=\n");
        for (int i = 0; i < 5; i++) {
            for (int j = 0; j < k; j++)
                printf("%7g ", D[i * k + j]);
            printf("\n");
        }

        delete[] I;
        delete[] D;
    }

    { // search xq
        idx_t* I = new idx_t[k * nq];
        float* D = new float[k * nq];

        index.search(nq, xq, k, D, I);

        printf("I (5 first results)=\n");
        for (int i = 0; i < 5; i++) {
            for (int j = 0; j < k; j++)
                printf("%5zd ", I[i * k + j]);
            printf("\n");
        }

        printf("D (5 last results)=\n");
        for (int i = nq - 5; i < nq; i++) {
            for (int j = 0; j < k; j++)
                printf("%5f ", D[i * k + j]);
            printf("\n");
        }

        delete[] I;
        delete[] D;
    }

    delete[] xb;
    delete[] xq;
    return 0;
}
```

</Tab>

</Tabs>

The sanity check searches the first 5 database vectors against themselves, so the first result for each should be the vector itself (distance 0).

</Step>

</Steps>

## Faster approximate search

`IndexFlatL2` scans every vector in the database — it gives exact results but does not scale to very large datasets. `IndexIVFFlat` partitions the database into `nlist` Voronoi cells and only searches the `nprobe` nearest cells at query time, making search much faster at a small accuracy cost.

```python
import numpy as np
import faiss

d = 64                           # dimension
nb = 100000                      # database size
nq = 10000                       # nb of queries
np.random.seed(1234)             # make reproducible
xb = np.random.random((nb, d)).astype('float32')
xb[:, 0] += np.arange(nb) / 1000.
xq = np.random.random((nq, d)).astype('float32')
xq[:, 0] += np.arange(nq) / 1000.

nlist = 100
k = 4
quantizer = faiss.IndexFlatL2(d)  # the other index
index = faiss.IndexIVFFlat(quantizer, d, nlist, faiss.METRIC_L2)
# here we specify METRIC_L2, by default it performs inner-product search

assert not index.is_trained
index.train(xb)
assert index.is_trained

index.add(xb)                  # add may be a bit slower as well
D, I = index.search(xq, k)     # actual search
print(I[-5:])                  # neighbors of the 5 last queries
index.nprobe = 10              # default nprobe is 1, try a few more
D, I = index.search(xq, k)
print(I[-5:])                  # neighbors of the 5 last queries
```

<Warning>
  IVF indexes must be trained on a representative sample of your data before vectors can be added. Call `index.train(xb)` before `index.add(xb)`.
</Warning>

Increasing `nprobe` improves recall at the cost of speed. `nprobe = 1` is the fastest but least accurate; `nprobe = nlist` is equivalent to an exact flat search.

## Using `index_factory`

The `index_factory` function constructs indexes from a string descriptor, avoiding the need to manually wire together quantizer and index objects:

```python
# Equivalent to the IVFFlat example above
index = faiss.index_factory(64, "IVF100,Flat")

# IVF with Product Quantization — more memory-efficient
index = faiss.index_factory(64, "IVF100,PQ8")
```

The string format is `"<preprocessing>,<index_type>"`. This is the recommended way to build complex composite indexes.

<Tip>
  Use `faiss.index_factory` when experimenting with different index types. It makes it easy to swap between `IVF100,Flat`, `IVF100,PQ8`, `HNSW32`, and others without changing surrounding code.
</Tip>
